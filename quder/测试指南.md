## 测试方案

### 方案 A: 本地完整测试 (需要后端)

**前提条件**: 后端服务运行在 localhost:8080

1. **启动后端服务**:
   ```bash
   cd backend
   mvn spring-boot:run
   ```

2. **启动前端开发服务器**:
   - 双击运行 `启动前端开发服务器.bat`
   - 或手动运行: `cd portfolio && npm run dev`

3. **访问地址**:
   - 首页: http://localhost:5173/
   - 管理后台: http://localhost:5173/admin

4. **测试功能**:
   - [ ] 多语言切换 (中文、English、日本語、한국어、Français、Español)
   - [ ] 页面滚动动画
   - [ ] 管理后台登录
   - [ ] 内容编辑与保存
   - [ ] WebSocket 实时同步 (打开两个浏览器窗口)

---

### 方案 B: 前端预览测试 (无需后端)

**适用场景**: 仅测试前端 UI,不测试后端 API

1. **启动前端开发服务器**:
   - 双击运行 `启动前端开发服务器.bat`

2. **访问地址**:
   - 首页: http://localhost:5173/
   - 管理后台: http://localhost:5173/admin

3. **预期行为**:
   - ✅ 首页可以正常显示 (使用本地数据)
   - ✅ 多语言切换正常
   - ⚠️ 管理后台保存功能会失败 (需要后端)

## 常见问题

### Q: 后端连接失败怎么办?
A: 前端已配置降级策略,会自动使用本地 localStorage 数据。

### Q: 如何测试实时同步?
A: 需要启动后端服务,打开两个浏览器窗口访问管理后台。

### Q: 生产环境如何切换后端 URL?
A: 修改 `.env.production` 中的 `VITE_API_URL` 和 `VITE_WS_URL`。
