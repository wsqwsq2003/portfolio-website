# 快速部署指南 - 5 分钟上线

## 🚀 开始部署

### 方法一：使用自动化脚本（推荐）

双击运行 `deploy.bat`，按照提示完成部署。

---

### 方法二：手动部署

#### 1️⃣ 部署后端到 Render（5分钟）

**打开浏览器访问：**
```
https://render.com
```

**步骤：**
1. 用 GitHub 账号登录
2. New + → Web Service
3. 连接仓库 `Wangshou123/qqqqq-demo`
4. 配置：
   - **Root Directory**: `backend`
   - **Build Command**: `mvn clean package -DskipTests`
   - **Start Command**: `java -jar target/*.jar`
   - **环境变量**：
     ```
     PORT=8080
     JAVA_OPTS=-Xmx512m
     ```
   - **健康检查**: `/api/portfolio/health`
5. Create Web Service
6. 等待部署完成

**复制后端 URL：**
```
https://portfolio-backend-xxxx.onrender.com
```

---

#### 2️⃣ 配置前端环境变量

在 `portfolio` 目录下创建 `.env.production` 文件：

```env
VITE_API_URL=https://portfolio-backend-xxxx.onrender.com/api
VITE_WS_URL=https://portfolio-backend-xxxx.onrender.com/ws
```

**提交到 Git：**
```bash
cd c:\Users\17988\Desktop\quder
git add portfolio\.env.production
git commit -m "update: 配置生产环境 URL"
git push
```

---

#### 3️⃣ 部署前端到 Netlify（2分钟）

**打开浏览器访问：**
```
https://netlify.com
```

**步骤：**
1. 用 GitHub 账号登录
2. Add new site → Import an existing project
3. 选择仓库 `Wangshou123/qqqqq-demo`
4. 自动识别配置（已存在 `netlify.toml`）
5. Deploy site

**获得网站 URL：**
```
https://your-site-name.netlify.app
```
