## 📦 第二步：部署后端到 Render

### 2.1 访问 Render

打开浏览器：https://render.com

### 2.2 登录/注册

- 点击 "Get Started for Free"
- 选择 "Sign in with GitHub"
- 授权访问你的 GitHub 账号

### 2.3 创建 Web Service

1. 点击 Dashboard 的 **"New +"**
2. 选择 **"Web Service"**
3. 选择 **"Connect a repository"**
4. 搜索并选择你刚创建的仓库（`portfolio-website`）
5. 点击 **"Connect"**

### 2.4 配置服务

Render 会自动读取 `render.yaml`，确认以下配置：

- **Name**: `portfolio-backend`
- **Region**: Singapore（推荐，离中国最近）
- **Branch**: `main`
- **Root Directory**: `backend`
- **Build Command**: `mvn clean package -DskipTests`
- **Start Command**: `java -jar target/*.jar`
- **Plan**: Free

### 2.5 环境变量

点击 "Advanced" → "Add Environment Variable"，添加：

```
PORT=8080
JAVA_OPTS=-Xmx512m
```

### 2.6 健康检查

- **Health Check Path**: `/api/portfolio/health`

### 2.7 创建服务

点击 **"Create Web Service"**

### 2.8 等待部署

- 构建时间：约 3-5 分钟
- 部署成功后，复制你的后端 URL：
  ```
  https://portfolio-backend-xxxx.onrender.com
  ```

## 🎨 第四步：部署前端到 Netlify

### 4.1 访问 Netlify

打开浏览器：https://netlify.com

### 4.2 登录

- 点击 "Sign in"
- 选择 "Sign in with GitHub"
- 授权访问

### 4.3 创建站点

1. 点击 **"Add new site"** → **"Import an existing project"**
2. 选择 **"GitHub"**
3. 找到并选择你的仓库（`portfolio-website`）
4. 点击 **"Connect"**

### 4.4 配置构建

Netlify 会自动读取 `netlify.toml`，确认：

- **Base directory**: 留空
- **Build command**: `cd portfolio && npm install && npm run build`
- **Publish directory**: `portfolio/dist`

### 4.5 环境变量（可选）

如果需要，可以在 Netlify Dashboard 添加：

- **Key**: `VITE_API_URL`
- **Value**: `https://portfolio-backend-xxxx.onrender.com/api`
- **Key**: `VITE_WS_URL`
- **Value**: `https://portfolio-backend-xxxx.onrender.com/ws`

### 4.6 部署

点击 **"Deploy site"**

### 4.7 等待部署

- 构建时间：约 1-2 分钟
- 部署成功后获得 URL：
  ```
  https://your-site-name.netlify.app
  ```

## 🔧 故障排除

### 后端部署失败

1. 查看 Render 构建日志
2. 确认 Java 版本正确（Java 21）
3. 检查端口配置为 8080

### 前端构建失败

1. 查看 Netlify 构建日志
2. 确认 Node.js 版本（18+）
3. 检查依赖是否安装

### WebSocket 连接失败

1. 检查 `VITE_WS_URL` 是否正确
2. 确认后端 CORS 配置
3. 查看浏览器控制台错误
